# `docs_v2`

This directory is the staging area for the future public-facing `src_v2`
documentation.

It is intentionally small and incomplete. The goal is to collect stable
breadcrumbs while the code and binary contracts are still moving, without
pretending that the documentation set is finished.

Current purpose:

- capture the major `src_v2` logic steps
- record the runtime and binary contracts
- keep design decisions close to implementation
- define quality gates before a larger Documenter/Literate site exists

Not the goal yet:

- polished user documentation
- exhaustive API reference
- tutorial notebooks
- stable installation or CLI guidance
